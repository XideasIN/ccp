Cookie System Integration

Complete guide for implementing the centralized cookie consent system

File Structure

The cookie consent system is organized as follows:

ReBuild/ ├── js/ │ └── cookie-consent.js # Main cookie consent script ├── cookie/ │ ├── cookie-demo.html # Interactive demo page │ └── cookie-integration-guide.html # This integration guide ├── wp-content/ │ └── cookie/ │ └── cookie.PNG # Cookie icon image └── privacy-policy/ └── index.html # Privacy policy page

Quick Integration

Follow these steps to add cookie consent to any page:

Include the Script

Add the cookie consent script to your HTML page:

<script src="js/cookie-consent.js"></script>

Automatic Initialization

The system automatically initializes when the page loads. No additional setup required!

Optional: Listen for Events

Add custom handling for consent changes:

window.addEventListener('cookieConsentChanged', function(event) { console.log('Consent status:', event.detail.choice); // Handle consent change (accepted, denied, dismissed) });

System Features

GDPR Compliance

Fully compliant with GDPR requirements including clear consent options and easy withdrawal.

10-Day Persistence

User consent is stored for 10 days using localStorage, reducing banner frequency.

Google Analytics Integration

Automatically manages Google Analytics consent based on user choice.

Responsive Design

Mobile-optimized with smooth animations and accessibility support.

API Reference

The cookie consent system provides several methods for programmatic control:

window.coralCookieConsent.getConsentStatus()

Returns the current consent status object or null if no consent exists.

const status = window.coralCookieConsent.getConsentStatus(); if (status) { console.log('Choice:', status.choice); // 'accepted', 'denied', or 'dismissed' console.log('Timestamp:', status.timestamp); // When consent was given console.log('Expires:', status.expiresAt); // When consent expires }
window.coralCookieConsent.resetConsent()

Clears stored consent, causing the banner to show again on next page load.

window.coralCookieConsent.resetConsent(); // Banner will appear on next page load or manual init()
window.coralCookieConsent.init()

Manually initialize the consent system (useful after reset).

window.coralCookieConsent.init(); // Shows banner if no valid consent exists

Customization

Styling

The cookie banner uses CSS custom properties and can be customized by overriding styles:

/* Override banner colors */ .coral-cookie-banner { --primary-color: #B1976B; --secondary-color: #0A3140; --accent-color: #007CBA; } /* Custom positioning */ .coral-cookie-banner { bottom: 10px !important; left: 10px !important; right: auto !important; }

Configuration

Modify the CookieConsentManager constructor to change default settings:

class CookieConsentManager { constructor() { this.COOKIE_NAME = 'your_custom_cookie_name'; this.EXPIRY_DAYS = 30; // Change expiry period // ... rest of constructor } }

Integration Examples

Basic Page Integration

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Your Page</title> </head> <body> <!-- Your page content --> <!-- Cookie consent script --> <script src="js/cookie-consent.js"></script> </body> </html>

With Custom Event Handling

<script src="js/cookie-consent.js"></script> <script> // Listen for consent changes window.addEventListener('cookieConsentChanged', function(event) { const choice = event.detail.choice; if (choice === 'accepted') { // Enable tracking scripts enableGoogleAnalytics(); enableFacebookPixel(); } else { // Disable tracking disableTracking(); } }); // Check existing consent on page load document.addEventListener('DOMContentLoaded', function() { const status = window.coralCookieConsent?.getConsentStatus(); if (status && status.choice === 'accepted') { enableGoogleAnalytics(); } }); </script>
Important: Ensure the privacy policy link in the cookie banner points to your actual privacy policy page. The default path is 'privacy-policy/index.html'.
